# Admin Module - Edit User Test Scenarios

## Feature
Edit System User

## Module
Admin

## Preconditions
- User is logged in as Admin.
- User exists in the System Users list.
- User is on the Admin > System Users page.

---

## Test Scenarios

### TS-ADMIN-EDIT-001
Verify Admin can edit an existing system user's status.

### TS-ADMIN-EDIT-002
Verify Admin can edit an existing system user's role.

### TS-ADMIN-EDIT-003
Verify changes are saved successfully.

### TS-ADMIN-EDIT-004
Verify updated information is displayed after saving.

### TS-ADMIN-EDIT-005
Verify Cancel button discards changes.

### TS-ADMIN-EDIT-006
Verify mandatory field validation during edit.

### TS-ADMIN-EDIT-007
Verify user remains searchable after update.

### TS-ADMIN-EDIT-008
Verify success notification is displayed after updating.
