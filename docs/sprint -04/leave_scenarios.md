# Leave Module - Manual Test Scenarios

## Sprint 4: Leave Module

### TS_Leave_01 - Verify Admin can create employee with login credentials
Validate that Admin can create a new employee and enable login credentials successfully.

### TS_Leave_02 - Verify Admin can add leave entitlement
Validate that Admin can add leave entitlement for the newly created employee.

### TS_Leave_03 - Verify Employee can login successfully
Validate that the newly created employee can login using generated credentials.

### TS_Leave_04 - Verify Employee can apply leave
Validate that Employee can apply leave after entitlement is added.

### TS_Leave_05 - Verify Employee can search applied leave
Validate that Employee can search and view the applied leave record in My Leave.

### TS_Leave_06 - Verify Employee can cancel applied leave
Validate that Employee can cancel the applied leave successfully.

### TS_Leave_07 - Verify cancelled leave status
Validate that cancelled leave displays the correct cancelled status.

### TS_Leave_08 - Verify complete Leave E2E workflow
Validate the complete workflow from employee creation to leave cancellation.
